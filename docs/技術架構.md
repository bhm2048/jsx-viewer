## 渲染管線詳解

### Step 1: 檔案讀取

```
使用者拖放 .jsx 檔案
        │
        ▼
DropZone.jsx
  ├── onDrop / onInputChange 事件
  ├── 過濾只接受 .jsx / .tsx 副檔名
  ├── FileReader.readAsText(file) 讀取內容
  └── addFile(name, content) 存入 store
```

### Step 2: Import 解析

### Step 3: Babel 編譯

```javascript
// RenderPane.jsx > compileJSX()

輸入（移除 import 後的原始碼）:
  export default function App() {
    return <div style={{ color: "red" }}>Hello</div>
  }

        │ Babel.transform(code, { presets: ['react'] })
        ▼

輸出（純 JavaScript）:
  function App() {
    return React.createElement("div", { style: { color: "red" } }, "Hello");
  }

同時捕獲:
  componentName = "App"
```

### Step 4: srcdoc 組裝

```html
<!-- template.js 產生的 HTML -->
<!DOCTYPE html>
<html>
<head>
  <script type="importmap">
    { "imports": { "react": "https://esm.sh/react@18.3.1", ... } }
  </script>
</head>
<body>
  <div id="root"></div>
  <script type="module">
    import React, { useState } from "react";
    import { createRoot } from "react-dom/client";

    try {
      // ← Babel 編譯後的程式碼插入這裡
      function App() {
        return React.createElement("div", { style: { color: "red" } }, "Hello");
      }

      const _root = createRoot(document.getElementById("root"));
      _root.render(React.createElement(App));

      window.parent.postMessage({ type: "render-success" }, "*");
    } catch(err) {
      window.parent.postMessage({ type: "render-error", message: err.message }, "*");
    }
  </script>
</body>
</html>
```

### Step 5: iframe 載入與通訊

```
Parent (RenderPane)              iframe
       │                            │
       │── srcdoc = html ──────────▶│
       │                            │── 解析 import map
       │                            │── 從 esm.sh 載入模組
       │                            │── 執行 compiled code
       │                            │── createRoot().render()
       │                            │
       │◀── postMessage ────────────│  (成功或失敗)
       │                            │
  setError() 或 clearError()        │
  顯示 ErrorPanel 或隱藏            │
```

## esm.sh 運作原理

```
瀏覽器看到: import { motion } from "framer-motion"
        │
        ▼
Import Map 對應: "framer-motion" → "https://esm.sh/framer-motion?external=react,react-dom"
        │
        ▼
瀏���器發送 HTTP 請求到 esm.sh
        │
        ▼
esm.sh 回傳:
  - 該套件的 ESM 格式版本
  - 自動解析所有子依賴
  - 但 react / react-dom 不打包（因為 ?external）
  - 這些 bare import 再由 Import Map 解析到統一的 React
        │
        ▼
所有套件共用同一份 React → 不會出現 "multiple React instances" 錯誤
```

## 已知限制（MVP）

| 限制 | 原因 | 未來改善方向 |
|------|------|------------|
| 不支援多行 import | 正則表達式限制 | 改用 AST parser |
| 不支援相對路徑 import | 無 bundler/FS | 加入多檔上傳與虛擬 FS |
| 不支援 TypeScript | Babel preset 未包含 | 加入 `@babel/preset-typescript` |
| 不支援 CSS Modules | iframe 無 bundler | 加入 CSS-in-JS 預處理 |
| 切換 tab 會重新渲染 | 每次替換整個 srcdoc | 使用多 iframe 快取 |
| 無檔案編輯功能 | MVP 範圍外 | 整合 CodeMirror |
