# JSX Viewer 操作手冊

## 啟動應用程式

### 首次使用

```bash
# 1. 進入專案目錄
cd d:\VSC-D\JSX

# 2. 安裝依賴（只需執行一次）
npm install

# 3. 啟動開發伺服器
npm run dev
```

瀏覽器開啟 `http://localhost:5173` 即可看到 JSX Viewer 介面。

### 日常啟動

```bash
cd d:\VSC-D\JSX && npm run dev
```

## 載入 JSX 檔案

### 方法一：拖放

1. 在檔案總管中找到你的 `.jsx` 檔案
2. 直接拖到左側「Drop .jsx files」區域
3. 檔案會立即出現在側邊欄並開始渲染

**支援同時拖放多個檔案。**

### 方法二：Browse 按鈕

1. 點擊拖放區下方的「Browse」按鈕
2. 在檔案對話框中選擇一或多個 `.jsx` / `.tsx` 檔案
3. 點擊「開啟」

## Tab 操作

| 操作 | 方法 |
|------|------|
| 切換檔案 | 點擊側邊欄的 Tab |
| 關閉檔案 | 點擊 Tab 右側的 `×` 按鈕 |
| 新增檔案 | 拖放或 Browse 載入新檔案 |

關閉目前檢視的 Tab 時，會自動切換到最近新增的其他 Tab。

## 第三方套件

JSX Viewer 會自動偵測你的檔案中使用的第三方套件，並從 [esm.sh](https://esm.sh) CDN 動態載入。

```jsx
// 這些 import 會自動從 CDN 載入：
import { motion } from "framer-motion";
import styled from "styled-components";
import { Chart } from "chart.js";
```

**注意事項：**
- 首次載入某個套件時需要網路連線
- 載入大型套件可能需要幾秒鐘（會顯示 Loading 指示器）
- `react` 和 `react-dom` 是內建提供的，不需要 CDN

## 錯誤處理

### 編譯錯誤（紅色面板）

如果 JSX 語法有誤，會立即顯示 Babel 的錯誤訊息：

```
Compilation Error
SyntaxError: Unexpected token (42:15)
```

**常見原因：**
- JSX 語法錯誤（未閉合的標籤、多餘的括號）
- 使用了 TypeScript 語法（目前 MVP 不支援 TS 編譯）

### 執行時錯誤

如果 JSX 語法正確但執行時出錯，iframe 內會顯示錯誤，同時 parent 會收到錯���通知。

**常見原因：**
- 第三方套件載入失敗（網路問題或套件不存在）
- 元件邏輯錯誤（存取 undefined 屬性等）

## JSX 檔案格式要求

### 必要條件

1. **必須有 default export：**
   ```jsx
   // 方式一：export default function
   export default function MyComponent() { ... }

   // 方式二：先定義後 export
   function MyComponent() { ... }
   export default MyComponent
   ```

2. **Import 必須是單行格式：**
   ```jsx
   // ✅ 支援
   import { useState, useEffect } from "react";

   // ❌ 不支援（多行 import）
   import {
     useState,
     useEffect
   } from "react";
   ```

3. **自包含（所有元件定義在同一檔案內）：**
   ```jsx
   // ❌ 不支援
   import Header from "./Header";  // 相對路徑 import 不會被解析
   ```

### Claude Code 產生的典型格式

Claude Code 產生的 JSX 說明文件通常符合以下模式，可直接使用：

```jsx
import { useState, useEffect, useRef } from "react";

// 子元件定義
function SubComponent({ prop }) {
  return <div>{prop}</div>;
}

// 主元件（有 export default）
export default function MainExplanation() {
  const [state, setState] = useState(initialValue);

  return (
    <div style={{ /* inline styles */ }}>
      <style>{`/* CSS animations */`}</style>
      <SubComponent prop="value" />
    </div>
  );
}
```

## 疑難排解

| 問題 | 可能原因 | 解決方法 |
|------|---------|---------|
| 拖放無反應 | 檔案不是 `.jsx` 或 `.tsx` | 確認副檔名正確 |
| 白屏無內容 | 元件回傳 null 或空內容 | 檢查元件邏輯 |
| 樣式異常 | 元件依賴外部 CSS 檔 | 確保所有���式寫在 JSX 內 |
| 套件載入失敗 | 網路問題或套件名稱拼錯 | 檢查 import 的套件名稱 |
| 「Babel not loaded」 | CDN 被阻擋或離線 | 確認可存取 unpkg.com |
